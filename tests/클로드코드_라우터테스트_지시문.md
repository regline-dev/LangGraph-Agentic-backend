# 클로드코드 작업 지시: 규칙 라우터 테스트 시나리오 실행

## 목적
아래 질문 목록을 실제로 하나씩 `/agent/chat`(또는 해당 엔드포인트)에 던져보고,
각 질문이 **어느 규칙에 걸렸는지 / 왜 그렇게 됐는지 / 기대와 다르면 무엇을 고쳐야 하는지**를
정해진 포맷으로 기록해줘.

## 기록 포맷 (반드시 이 형식 그대로)

```
N. <질문> : <정상 | 기대이하>
   [규칙] <실제로 매칭된 규칙 이름과 근거 코드 위치>
   [원인] <왜 그 규칙에 걸렸는지, 정상이면 "제목+트리거 매칭" 같은 이유>
   [해결] <기대이하일 때만 작성 — 목록 확장이 아니라 정규식/패턴 매칭 방식으로 제안할 것>
   [상태] pass | x (해결 대기)
```

## 판단 기준
- 응답에 `[근거] ...pdf p.N`이 붙어있으면 → 검색 경로(Agent 루프)를 탄 것
- 유사도 값이 있으면 → 검색 경로, 없으면 → 규칙 라우터(metadata/verbatim) 경로
- "기대이하"로 판단하는 기준: 원문/지표 요청인데 검색 경로를 탔거나, 명백한 의도(예: MBTI 변경 요청)인데 일반론 답변이 나온 경우
- 해결책 제안 시 **키워드 목록 단순 추가는 지양** — 정규식 패턴(예: `mbti` + `(바꾸|설정|수정)`)처럼 변형 표현까지 포괄하는 방식을 우선 제안

## 테스트 질문 목록

**정상 케이스 (1개씩만)**
1. "늑대와 어린양 내용평가는"
2. "박쥐와 족제비 원문 보여줘"
3. "아버지와 아들들 줄거리 알려줘"

**애매/경계 케이스**
4. "박쥐"
5. "해석"
6. "결론"
7. "재미도"
8. "내용평가는" (대화 시작하자마자, 맥락 없음)

**여러 우화 상황 (단기 메모리 맥락 확인)**
9. "늑대와 어린양 얘기하고" → 다음 턴 "내용평가는"
10. "박쥐와 족제비 얘기하고" → 다음 턴 "원문 보여줘"
11. "제일 재밌는 우화가 뭐야"
12. "결말톤이 새드인 우화 있어?"

**MBTI 관련**
13. "박쥐와 족제비 한마디 결론" (MBTI 미설정)
14. "MBTI : enfp"
15. "박쥐와 족제비 유형별 해석" (14번 이후, MBTI 있음)
16. "mbti 변경"
17. "MBTI : 그냥" (잘못된 형식)
18. "mbti로 바꿔 보려면" ← 실사용에서 기대이하로 확인된 케이스
19. "mbti 수정" ← 실사용에서 기대이하로 확인된 케이스

**원문 vs 한마디결론 혼동 유도**
20. "아버지와 아들들 오늘날로 치면" (하위호환 라벨)
21. "아버지와 아들들 그대로 읽어줘" ("원문"이라는 단어 없이 원문 요청)

## 산출물
- 위 21개 전부 실행한 결과를 하나의 마크다운 표/리스트로 정리
- `[상태] x`인 것들은 별도 섹션으로 모아서 "다음에 고칠 것" 목록으로 제시
- 새로 발견된 패턴(목록에 없던 표현인데 기대이하로 나온 것)이 있으면 추가로 기록
